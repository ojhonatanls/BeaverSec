# Security Policy

## Supported Versions

This section informs which versions of BeaverSec are actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 4.x.x   | :white_check_mark: |
| < 4.0   | :x:                |

Version 4.0 represents a complete refactoring with a strong focus on security. All users are encouraged to upgrade to the latest version.

## Reporting a Vulnerability

Security is a top priority for BeaverSec. If you discover a vulnerability, please help us address it responsibly.

**How to report:**

1.  **Do not open a public issue.** This could expose the problem before we have a chance to release a fix.
2.  Send the details of the vulnerability via email to: **jhonatanlucassantosofc@gmail.com**.
3.  Alternatively, you can open a **Security Advisory** directly on GitHub. This is a secure, private channel for discussing vulnerabilities.

**What to expect after reporting:**

*   **Acknowledgment:** We will confirm receipt of your report within 48 hours.
*   **Analysis:** We will work to validate and understand the impact of the vulnerability.
*   **Update:** We will keep you informed about the progress of the fix.
*   **Recognition:** We appreciate your contribution to the security of the project. With your permission, your name will be included in the acknowledgments.

Thank you for helping keep BeaverSec secure for everyone.
